3D as Code Roadmap

Text became the universal interface for software.
3D is the universal interface for space.

Designing a physical product — a speaker, a phone, a medical device — means juggling shape, heat, cost, and manufacturing across half a dozen specialists and tools. AI can help with each piece separately, but it can't yet see the product itself. Give an AI eyes on the 3D model, and it can reason across all of them at once.

Code

Medium: Text — instructions, logic, programs
Properties: Inspectable, editable, versionable, composable
AI unlock: LLM Code — the LLM sees your repo

3D

Medium: Geometry — shapes, assemblies, worlds
Properties: Inspectable, editable, versionable, composable
AI unlock: LLM 3D — the LLM sees in 3D, and you point back
1

3D is the spatial equivalent of code

  • Physical products already live in 3D files. Shape, fit, tolerance, and assembly all exist as 3D models — the source of truth that designers, engineers, and manufacturers share. AI just can't read them yet.
  • Same playbook as code. When AI got access to code repositories, it could finally help build software end-to-end. 3D files are the equivalent for physical things — structured, inspectable, editable.
  • It needs to be two-way. The AI should be able to point at parts, highlight clashes, and propose changes. The user should be able to point back — "this surface, not that one." The 3D model becomes a shared workspace.
2

What this looks like in practice

  • Try the demo. Open the home page and click "Case 1: ID wants 30% smaller." The setup: industrial design wants a smart speaker shrunk by 30%. The AI checks if the new shape causes overheating, flags parts that no longer fit, checks the power budget, and estimates the cost change — pulling from design, simulation, and supply chain in one answer.
  • That cross-domain reasoning takes a room of specialists today. Mechanical, thermal, electrical, manufacturing, supply chain — each looks at one slice and emails the others. The AI can hold all of them in context at once.
  • And it's not just for engineers. 3D today is fragmented — dozens of file formats, tools that cost thousands and take years to learn. Plain-language access to 3D could open spatial design to people who've never been able to use it.
3

Why I think it's tractable

  • Today's AI can already reason about geometry. I tested a leading model on GeoGramBench, a public set of geometry-reasoning problems — it scored 97.6% (488 of 500). The reasoning is there. What's missing is access to real product files.
  • Tools, not bigger models. AI got good at code when it got access to a few simple tools — read a file, search a folder. The same playbook should work here: a small set of "spatial tools" (measure this, find that part, check this clearance) gives the AI a way in.
  • I work on this every day. I use AI coding tools daily in hardware R&D at Apple. They help with isolated software problems, but I'm hitting the ceiling of what isolated tools can do. The next step is giving the AI eyes on geometry.
  • Product design is the right place to start. Well-defined workflows, structured data, and tight feedback loops. Architecture, robotics, and digital twins follow naturally.
