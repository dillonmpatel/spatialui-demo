Phase 1: Baseline Established

Weeks 1–6

I ran a leading LLM on GeoGramBench to establish a spatial reasoning baseline and it had an 97.6% accuracy (488/500). While I haven't benchmarked against the latest competitor performance, LLMs are capable of interpreting code-described geometry.

Remaining Phase 1 work: Build Spatial MCP + measure tool delta
  • Build minimal Spatial MCP schema: select_region, measure_distance, query_feature_tree.
  • Re-run GeoGramBench and 3DSRBench with tool access. Measure the delta on real CAD assemblies (not just procedural geometry code).
  • GeoGramBench tests code-described geometry. Real CAD files are harder — tool access could bridge that gap. If the LLM Code analogy holds — where tool access dramatically improved code task performance — a 40-60% improvement for spatial tools would be a reasonable initial hypothesis.
Objective: Quantify tool-augmented improvement on real CAD data.
KR1: Establish zero-shot baseline on 3DSRBench assembly tasks.
KR2: Measure tool-augmented delta on same tasks — hypothesis: ≥40% gap closure.
KR3: Identify categories where tools alone are insufficient and fine-tuning is needed.

Phase 2: Close the Gap

Weeks 7–10

SpatialVLM demonstrated that synthetic data generation can unlock metric spatial reasoning — 2B VQA examples from 10M images.5 CAD-Llama showed that CAD-as-code fine-tuning can achieve 0.966 command accuracy.2 This phase adapts both approaches.

  • Generate synthetic CAD training data following the SpatialVLM pipeline: CAD viewport screenshots paired with scene graph ground truth.5
  • Synthetic GD&T annotation pairs — tolerance callouts matched to geometry features.
  • Multi-view renders of assemblies with feature tree labels for visual grounding.
  • Tolerance chain VQA pairs — "what is the stack-up from datum A to surface B?"
  • Fine-tune using SpatialLLM recipe: 3D-informed multimodal alignment + visual instruction tuning. Avoid fine-tuning the visual encoder directly (hurts generalization).6
Objective: Close the gap between benchmark geometry and real-world CAD reasoning.
KR1: Generate synthetic CAD training dataset (≥10K viewport + scene graph pairs).
KR2: Fine-tuned accuracy approaches CAD-Llama's 0.966 command accuracy on assembly feature ID.
KR3: Feature ID accuracy >0.90 on 3DSRBench-style assembly QA.

Phase 3: Ship and Dogfood

Weeks 11–12
  • Run the full eval suite across all three tiers (zero-shot, tool-augmented, fine-tuned) and publish an internal benchmark.
  • Get a dogfood version in front of real users — product designers, mechanical engineers, program managers — and collect feedback on where it works and where it falls apart.
  • Build a tolerance chain reasoning eval. Nothing like this exists today, and it would be a proprietary benchmark Anthropic owns.
  • Document where tools alone are enough and where fine-tuning is actually needed.
Objective: Validate whether a tool surface can meaningfully close the gap without full fine-tuning.
KR1: Internal benchmark published across all three tiers.
KR2: Dogfood shipped to real users with at least 2 validated workflows.
KR3: Clear documentation of tool-only vs fine-tuning-required categories.

Four Research Gaps — Parallelizable

Independent workstreams

These workstreams are independent and can run concurrently. None requires the others to be complete before starting.

3D Vision Fine-Tune CAD viewport → scene graph training pairs. Synthetic GD&T annotations. Following SpatialVLM's data generation pipeline. Key risk: fine-tuning visual encoders with 3D data can hurt generalization.5,6
Spatial MCP Schema Tool definitions for select_region, measure_distance, query_feature_tree, set_section_plane (read), and modify_geometry, create_feature (write). Auth model for CAD file access. Geometry context protocol.
CAD Encoder Format-agnostic geometry compiler: mesh / point cloud / STEP / STL / OBJ / USD → canonical JSON/code. Aims to recover design intent (primitives, features, holes, fillets) from raw triangles. Builds on cadrille and Point2CAD research. Could enable non-CAD domains (3D art, architecture, robotics) where native formats are meshes, not parametric CAD.7,9
Token-Efficient Geometry The back half of the encoder pipeline: once geometry is in canonical form, compress it for context. 50-part feature trees can exceed 100K tokens. Hierarchical summarization preserving tolerance-critical features while discarding cosmetic detail. Follows CADmium's minimal-JSON approach.1

Full pipeline: Raw 3D (any format) → CAD Encoder → Canonical JSON/code → Token compression → Context window

Deployment Tiers — RSP Constraint

3 tiers

Reading ITAR defense CAD or weapons manufacturing geometry may trigger elevated capability thresholds under Anthropic's Responsible Scaling Policy. The Spatial MCP schema must account for deployment tier from day one.

Commercial
Standard Anthropic API. Consumer electronics, automotive, medical devices.
GovCloud / FedRAMP
Government-compliant infrastructure. Defense subcontractors, aerospace. ITAR-aware.
Air-Gapped On-Prem
Fully isolated deployment. Classified programs. No external data egress.
References
  1. CADmium — Mila (Dec 2025). Fine-tuned Qwen2.5-Coder on minimal-JSON CAD histories. Code LLMs naturally handle structured CAD formats.
  2. Li et al. — CAD-Llama (CVPR 2025). 99.9% unconditional generation success. 0.966 command type accuracy on structured CAD code.
  3. GeoGramBench (NeurIPS 2025 submission). 500 problems testing Program-to-Geometry translation. <50% at highest abstraction, >80% on local primitives.
  4. Ma et al. — 3DSRBench (2024). 2,762 3D spatial reasoning questions across 12 subtypes. SOTA LMMs show degraded performance on uncommon viewpoints.
  5. SpatialVLM — Google (2024). 2B VQA examples on 10M images. Proved synthetic data generation unlocks metric-scale spatial reasoning.
  6. SpatialLLM (CVPR 2025). 3D-informed multimodal alignment. SOTA on 3DSRBench. Finding: fine-tuning CLIP with 3D data hurts generalization.
  7. cadrille (ICLR 2026). First unified multimodal CAD reconstruction. RL fine-tuning on procedural data outperforms SFT on handcrafted datasets.
  8. IJCAI 2025 Survey. Comprehensive taxonomy of spatial reasoning approaches (image-based, point cloud-based, hybrid).
  9. Point2CAD / Point2Sequence. Reconstructs B-rep CAD from point clouds by predicting sketch+extrude sequences. Mesh → point cloud → parametric CAD pipeline.
  10. BrepGen (2024). Generates B-rep CAD directly including topology (faces, edges, vertices with NURBS parameters). Candidate target representation for CAD encoder output.
  11. Xu et al. — CAD-MLLM (2024). Unified encoder for text, multi-view images, and 3D point clouds. Frozen vision/point encoders with trainable linear projections into LLM.
